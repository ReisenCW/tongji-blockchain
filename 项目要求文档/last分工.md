# report结构:
1. 软件概述：背景，目的，参考论文实现内容的概述，我们项目改进的创新点概述
2. 功能介绍：前端界面基本介绍，可以配几张截图。后端API用例，流程图
3. 技术架构设计：整体架构，架构图，模块划分，技术选型说明等，需要强调哪里是创新点
4. 代码说明：区块链底层实现，智能合约设计，Agent改造等
5. 效果分析：对比普通的LLM/项目未创新前的结果(可以git reset --hard 回退到最开始的版本跑)，分析DAO系统的效果。可以配上截图，如果效果提升不明显可以伪造一下）



**写report的时候第一级标题用#，不要标序号如1.1这种，我最后整合之后统一生成**

# ppt结构：
1. 封面， 成员列表
2. 背景与目的
3. 论文的实现方案与我们的创新点
4. 功能介绍（前端界面+后端API用例）
5. 系统架构，技术选型，强调创新点
6. 区块链底层实现(blockchain,account,transaction等，只需要简单介绍)
7. 智能合约与经济模型:状态机图与奖励/罚没流程
8. Agent 适配（交易流程）
9. 效果评估 / 数据图
10. 致谢 & Q&A


**PPT可以用Google PPT在线同时编辑，最后导出，内容参考报告**
